# Finance Digest

A daily IB-grade market briefing and a LinkedIn editorial series, generated by a single Python script and a GitHub Actions cron. Free, fully auditable, runs in under a minute, costs $0.

Live archive: [subhankarshukla04.github.io/finance-digest](https://subhankarshukla04.github.io/finance-digest)

## What it does

Every weekday at 9:30 PM ET, after the US market close:

1. Pulls Yahoo Finance closing data for ~17 instruments — equity indices, US/Canada rates, FX majors, oil, gold, copper, BTC.
2. Pulls market news from a curated set of RSS feeds (Economist, Reuters, CNBC, MarketWatch, Financial Post, BNN, etc.).
3. Generates the daily digest (HTML, IB-style: hero board → secondary table → top stories → "edge of the day").
4. Picks one story for the LinkedIn editorial series, ghostwrites a post in my voice, and pushes it via Bark to my phone for review before posting.
5. Commits the digest to `docs/`, updates `chronicle.json`, and pushes — the GitHub Pages archive updates itself.

The "edge of the day" rotates through a fixed weekly lane:

| Day | Topic |
|---|---|
| Mon | Private Credit Boom |
| Tue | AI Capex vs. ROI |
| Wed | Canadian Economy & Bank of Canada |
| Thu | Sovereign Debt Stress |
| Fri | Deglobalization & Supply Chains |

Same five lanes for the LinkedIn post, picked in rotation with anti-repeat logic against the last 20 entries in `chronicle.json`.

## Why it exists

Two reasons:

1. **Discipline.** If I want a job in markets, I need to read markets every single day, not just when I feel like it. A cron job is harder to skip than a habit.
2. **Auditability.** A year from now, I want a public record of what I was tracking, what I thought was important, and how my reads aged. The `chronicle.json` is the receipts.

The LinkedIn series is "i saw this, here's how i'm thinking about it" — discovery-driven, not authority-driven. I'm a 3rd-year student. The frame matches the credential.

## Stack

- **Python 3.11** — `feedparser`, `requests`, no LLM SDK (raw HTTP to Groq for free Llama / DeepSeek inference)
- **Groq API** for the editorial pass (free tier; Llama 3.3 70B)
- **Yahoo Finance** for prices (no API key, scraped via the v8 chart endpoint)
- **GitHub Actions** for the daily cron (`daily-digest.yml`)
- **GitHub Pages** for the public archive
- **Bark** (iOS push notification service) for the 9:30 PM PT delivery to my phone

No paid APIs. No paid hosting. The whole thing runs on free tiers and a single 1,100-line script.

## Files

```
digest.py              # the entire pipeline
requirements.txt       # feedparser, requests
docs/
  YYYY-MM-DD.html      # one digest per market day
  chronicle.json       # LinkedIn series log (post #, date, category, hash)
  index.html           # archive landing page
.github/workflows/
  daily-digest.yml     # cron + auth
```

## Run locally

```bash
pip install -r requirements.txt
export GROQ_API_KEY=...
export NEWS_API_KEY=...   # optional; falls back to RSS only
export BARK_KEY=...       # optional; for phone push
python digest.py
```

Output lands in `docs/` as `YYYY-MM-DD.html`.

## Editorial rules (locked)

- No politics, no war coverage. Markets-only lens.
- Links go in the post body, not in comments.
- One story per day. Anti-repeat against the last 20 posts.
- I review every LinkedIn draft on my phone before it goes out — the cron writes, I publish.
